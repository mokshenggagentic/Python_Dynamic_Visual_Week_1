Lesson 04 · Speed & Scale

NumPy — Numbers in parallel, at scale

When your dataset is a million rows, plain Python loops are too slow. NumPy lets you do maths on entire columns at once — vectorised, parallel, and dramatically faster. This lesson covers the three superpowers every analyst uses daily.

The Parallel Calculator

Vectorised Operations — All cells at once

A Python for-loop processes one number at a time. NumPy processes the entire array in one shot, using highly-optimised C code under the hood. The difference on real datasets is 50–200× — minutes become seconds.

Industry Scenario
An e-commerce finance team needs to add 18% GST to 10 million order amounts before filing tax returns. Using a Python loop, the script ran for 12 minutes. After switching to a single NumPy expression: under 0.3 seconds.

vectorise.py
import numpy as np import time prices = np.array([120, 85, 240, ...]) # 10M items # ❌ The slow way — Python loop t = time.time() result_slow = [] for p in prices: result_slow.append(p * 1.18) print(time.time() - t) # ~12 sec # ✅ The fast way — one NumPy line t = time.time() result_fast = prices * 1.18 print(time.time() - t) # ~0.08 sec
Real benchmark, 10 million prices: Python loop : 12.4 s NumPy vectorised : 0.08 s → 155× faster, same result Why? NumPy doesn't loop in Python. It hands the whole array to compiled C code that uses CPU SIMD instructions to process many numbers per clock cycle. Bonus operations that vectorise: + - * / ** % np.log np.exp np.sqrt comparisons (>, ==, &, |)
Key takeaway
If you find yourself writing for x in column: ... on a numeric column, stop. There is almost certainly a vectorised NumPy expression that does the same thing 50–200× faster and reads as a single line.

The Number Summariser

Statistical Functions — Summarise an entire array in one call

NumPy gives you single-call versions of every stat you'll ever need: .sum(), .mean(), .median(), .std(), .min(), .max(). Each is vectorised. Each runs in milliseconds.

Industry Scenario
A retail analyst at DMart is looking at daily revenue for 10 days. The CFO wants to know: average daily revenue, the median (less affected by outliers), and how volatile sales were (standard deviation). One NumPy call per question.

stats.py
import numpy as np revenue = np.array([42, 58, 51, 73, 46, 61, 68, 55, 82, 49]) # Each stat is a single method call revenue.sum() # 585 → total over 10 days revenue.mean() # 58.5 → average daily revenue np.median(revenue) # 56.5 → robust to outliers revenue.std() # 12.0 → daily volatility revenue.min() # 42 → worst day revenue.max() # 82 → best day # Want them all together? np.describe(revenue)
Quick CFO briefing: total ......... ₹5,85,000 mean .......... ₹58,500/day median ........ ₹56,500/day std ........... ₹12,000 min ........... ₹42,000 (Day 1) max ........... ₹82,000 (Day 9) Analyst insight: mean > median → upper outlier (the big day 9 lifts the average above the typical day)
Key takeaway
Statistical functions are NumPy's summary toolkit. They turn an array of 10 (or 10 million) numbers into a single insight. Pay attention to mean vs median — when they diverge, you have a skewed distribution (outliers pulling the average).

The Direction Dial

Column-level Calculations — Choosing your axis

Real datasets are 2D — rows and columns. NumPy lets you aggregate along a chosen axis. axis=0 collapses rows (gives you per-column totals), axis=1 collapses columns (gives you per-row totals). One mental model unlocks all 2D analysis.

Industry Scenario
A regional manager at Big Bazaar has sales data for 4 stores across 4 months. Two questions: (a) Which month was best across all stores? (b) Which store was top this quarter? Same data, two axes.
Pick an axis — watch the totals form
Ready · axis=0
axis=0 means "collapse along rows" → you end up with one value per column. Useful when columns are months: gives you total per month.

axis.py
import numpy as np # 4 stores × 4 months sales = np.array([ [42, 51, 48, 55], # Mumbai [38, 45, 52, 49], # Pune [61, 58, 66, 70], # Delhi [29, 35, 41, 38], # BLR ]) # axis=0 → total PER MONTH (down columns) sales.sum(axis=0) # → [170, 189, 207, 212] ← Apr was best # axis=1 → total PER STORE (across rows) sales.sum(axis=1) # → [196, 184, 255, 143] ← Delhi led # Same idea works with mean, max, std… sales.mean(axis=1)
Monthly totals (axis=0): Jan: 170 Feb: 189 Mar: 207 Apr: 212 ★ peak Store totals (axis=1): Mumbai: 196 Pune : 184 Delhi : 255 ★ top performer BLR : 143 Insight (axis=0): Sales are growing month-over-month → seasonality kicking in. Insight (axis=1): Delhi outperforms by 30% → study what they're doing differently.
Key takeaway
Every aggregation answer comes from "which direction am I summarising?" — axis=0 collapses rows into one value per column, axis=1 collapses columns into one value per row. Memorise this once and every .sum(), .mean(), .max() on 2D data follows.
