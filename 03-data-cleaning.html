The Empty-Cell Problem

Missing Values — Decide before you analyse

Real datasets always have gaps. A customer didn't fill in their age, a sensor missed a reading, an export failed mid-file. These show up as NaN (Not a Number). You have three main strategies — and choosing the right one matters.

You're analysing a Flipkart customer table. Some rows are missing the customer's age or last purchase amount. You need to decide: drop those customers, or fill in the gaps so the totals still make sense.

import pandas as pd df = pd.read_csv('customers.csv') # Strategy 1 — drop rows with any NaN df1 = df.dropna() # Strategy 2 — fill numeric NaN with the column mean df2 = df.fillna({ 'age': df['age'].mean(), 'last_amt': df['last_amt'].mean() }) # Strategy 3 — forward-fill (use previous row's value) df3 = df.ffill()
When to use what
▸ dropna() — safest when only a few rows are missing and you have plenty of data. Risk: losing too many rows.

▸ fillna(mean) — good for symmetric numeric data. Risk: shrinks variance, blurs outliers.

▸ ffill() — perfect for time-series (yesterday's price is a fair guess for today). Risk: not valid for unordered data.
Key takeaway
Missing values are decisions, not bugs. The wrong strategy can mislead an entire dashboard — drop too much and you lose signal, fill carelessly and you invent it. Always inspect df.isna().sum() before deciding.

The Twin Problem

Duplicate Detection — Catch the same record twice

A user signed up twice. A sales rep accidentally double-entered an order. A CSV got merged with itself. Duplicates inflate metrics and skew models. Pandas gives you fine-grained control over how "duplicate" is defined.

Industry Scenario
You're cleaning a marketing email list for Swiggy. Some customers appear with the same email but different signup dates — you want to keep only the most recent signup per email.

dedupe.py
# Find duplicates (exact match across ALL columns) df.duplicated().sum() # Find duplicates by a SUBSET of columns df.duplicated(subset=['email']).sum() # Drop duplicates — keep the LATEST signup per email df_clean = ( df.sort_values('signup_date') .drop_duplicates( subset=['email'], keep='last' ) )
Before: 5 rows Duplicates on email: 2 After cleanup → 3 rows ✓ asha@gmail.com (2024-05-20) ✓ priya@gmail.com (2024-03-10) ✓ rohit@yahoo.com (2024-06-12) Dropped: ✗ id=1 asha@gmail.com (older) ✗ id=2 rohit@yahoo.com (older)
Key takeaway
"Duplicate" is a business question, not a technical one. Same row? Same email? Same name+phone? Define it explicitly with subset= and decide whether to keep the first, last, or aggregate them.

The Wrong-Dressed Data

Type Conversion — Numbers stuck inside strings

A price column shows "₹1,500" — looks like a number, but Python sees it as text. You can't sum it, you can't average it, you can't plot it. This lesson shows the cleanup pipeline that turns text-shaped numbers into real numbers.

Industry Scenario
A finance team exported a sales report from SAP where the amount column was saved as text — complete with currency symbols and comma separators. You need to turn it into a numeric column you can sum and average.

convert_types.py
# Before: amount column dtype is 'object' (text) df['amount'].dtype # → dtype('O') # Chain string methods, then cast df['amount'] = ( df['amount'] .str.replace('₹', '') .str.replace(',', '') .astype(float) ) # For safety against junk values, use pd.to_numeric df['amount'] = pd.to_numeric( df['amount'], errors='coerce' )
df.dtypes (before) customer_id object amount object ← problem! date object df.dtypes (after) customer_id object amount float64 ✓ date object Now you can: df['amount'].sum() → 4,820.0 df['amount'].mean() → 964.0 df['amount'].plot() ✓
Key takeaway
Always check df.dtypes early. If a column you expect to be numeric shows as object, something is hiding in there — currency symbols, commas, stray spaces. Use pd.to_numeric(errors='coerce') to gracefully convert and turn unparseable values into NaN (which you then handle with strategy 1).

The Time Decoder

Date Parsing — Unlock the calendar inside a string

Dates often arrive as plain text — "2024-03-15" or "15/03/2024". Parse them into proper datetime objects and you unlock filtering by month, day-of-week, time-of-day — the entire calendar.

Industry Scenario
An e-commerce team wants to know: Which day of the week brings the most orders? The dataset has a transaction_date column as text. Until you parse it to datetime, you can't extract weekday, group by month, or chart trends.

# Parse the column (handles many formats automatically) df['date'] = pd.to_datetime( df['date'], errors='coerce' ) # Now extract anything you need df['year'] = df['date'].dt.year df['month'] = df['date'].dt.month df['weekday'] = df['date'].dt.day_name() df['quarter'] = df['date'].dt.quarter df['is_weekend'] = df['date'].dt.dayofweek >= 5 # Filter — much easier with proper dates df_q1 = df[df['date'].dt.quarter == 1] df_recent = df[df['date'] >= '2024-01-01']
"Which weekday brings the most orders?" orders.groupby('weekday')['amt'].sum() Saturday ₹2,84,500 ★ peak Sunday ₹2,61,200 Friday ₹1,98,400 Thursday ₹1,52,300 Wednesday ₹1,41,800 Monday ₹1,38,600 Tuesday ₹1,29,100 Insight: Run weekend promotions.
Key takeaway
A parsed datetime column is the gateway to time-aware analysis — cohort retention, seasonality, day-of-week effects, year-over-year comparisons. Always parse dates early; never store them as plain strings in a working dataset.

The Untidy Text

String Cleaning — Same name, written six ways

Customer-entered data is wild: " Asha SHARMA ", "asha sharma", and "ASHA Sharma" are all the same person — but a computer sees three different strings. A short cleaning pipeline normalises them.

Industry Scenario
A CRM team is trying to count unique customers but the same person appears with inconsistent spacing and capitalisation. After cleaning, three rows collapse into one customer.
Watch four operations clean the strings
1. strip() 2. lower() 3. collapse spaces 4. title()

clean_strings.py
# Chain all cleaning ops together df['name_clean'] = ( df['name'] .str.strip() .str.lower() .str.replace( r'\s+', ' ', regex=True ) .str.title() ) # How many real unique customers? df['name'].nunique() # 3 (raw) df['name_clean'].nunique() # 1 (cleaned!)
BEFORE cleaning → 3 distinct values " Asha SHARMA " "asha sharma" "ASHA Sharma" AFTER cleaning → 1 unique customer "Asha Sharma" Business impact: Before: 3 separate customer profiles After: 1 unified profile Accurate revenue, accurate CRM.
Key takeaway
String cleaning is the difference between "3 customers worth ₹500 each" and "1 customer worth ₹1,500". The chain strip → lower → collapse spaces → title handles 90% of customer-entered data. Build it once, apply it to every text column.
