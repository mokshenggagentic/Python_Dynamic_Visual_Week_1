The Reusable Machine

Functions — Recipes that turn inputs into outputs

A function is a named block of instructions. You feed it ingredients (called arguments), it follows your recipe, and hands back a result. Once written, you can call it a hundred times without rewriting the logic.

Industry Scenario
An e-commerce analyst needs to calculate the total price after tax for every order. Instead of doing the math by hand for 50,000 orders, she writes a function once and reuses it.

Without functions, you'd paste the same math everywhere. Change the formula once, and every report instantly uses the new version. Functions are the foundation of clean, reliable data pipelines.

The One-Line Filter

List Comprehensions — Filter and transform in one line

Imagine going through a list of 1,000 sales transactions and picking only the ones above ₹500, while also rounding their amounts. List comprehensions do this in a single readable line, the way data analysts love it.

Industry Scenario
A finance team has daily transaction amounts. They need to identify high-value orders above ₹500 for their VIP rewards program — and apply a 10% loyalty bonus to each.

vip_bonus.py
transactions = [120, 750, 300, 980, 450, 1200] # Keep > 500 AND apply 10% bonus vip_rewards = [ round(amt * 1.1, 2) for amt in transactions if amt > 500 ] print(vip_rewards)

💡 Why this matters
List comprehensions replace 5–10 lines of loop code with one line. They're also 30–40% faster than regular loops in Python. Every senior analyst writes them daily.

The Instant Lookup

Dictionaries — Find any record by its label

A dictionary stores data as key → value pairs. Want a customer's email by their ID? Their loyalty tier? Their last order? Instead of scanning thousands of rows, a dictionary jumps straight to it.

Industry Scenario
A SaaS support team wants to instantly look up a customer's plan, region, and email from their user ID — without searching through a giant CSV every time.
Watch a key unlock its value

customer = { "name": "Priya Sharma", "plan": "Pro Annual", "region": "Mumbai", "email": "priya@nova.co", "mrr": 2400, } # Look up any field instantly print(customer["region"]) print(customer["plan"]) # Add or update on the fly customer["last_login"] = "2026-05-20"
Output
Mumbai Pro Annual # A dict with 1,000,000 entries # still returns a value in microseconds.
💡 Why this matters
Lists are ordered, but dictionaries are labelled. When data has names (customer IDs, product codes, country codes), dictionaries are massively faster than scanning a list — even on millions of records.

The Safety Net

Error Handling — Code that doesn't crash on bad data

Real-world data is messy. Files go missing, numbers arrive as text, dates are blank. try/except tells Python: "if something breaks, here's what to do instead of crashing the whole program."

Industry Scenario
A nightly ETL job reads sales CSVs from 20 stores. If one file is corrupted, you want the other 19 to still process — not the whole pipeline to fail.
Watch try/except catch a problem
Step 0 · Ready
We'll try to convert "N/A" to a number. Without protection: crash. With try/except: graceful recovery.

safe_loader.py
sales_raw = ["450", "N/A", "890", "", "320"] clean_sales = [] for value in sales_raw: try: clean_sales.append(float(value)) except ValueError: # Replace bad value with 0 clean_sales.append(0.0) print(f"Skipped: '{value}'") print(clean_sales)
Output
Skipped: 'N/A' Skipped: '' [450.0, 0.0, 890.0, 0.0, 320.0] # Without try/except, the program # would CRASH on the second value.
💡 Why this matters
In production, data pipelines must be resilient. A single bad row shouldn't take down a job that processes a million rows. try/except is the #1 tool for production-grade Python.
